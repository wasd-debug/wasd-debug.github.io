Git

git config –global user.email ……

git config –global user.name “…”

设置姓名和邮箱

git init:创建本地仓库

包含.git文件夹的文件夹就是工作目录

git status：查看仓库状态

git add . :添加工作区所有文件到暂存区

git add 文件名：添加文件至缓存区

git commit -m “描述信息”

git remote add “仓库名” “仓库网址” 远程仓库地址关联本地仓库

git remote -v： 查看远程仓库的信息

git push “仓库名” “分支名” ：将本地仓库推到远程仓库

git clone “地址”：从远程仓库克隆代码，并自动形成一个本地仓库

git pull “仓库名” “分支名”：拉取更改

git branch:查看分支

一般主分支中是稳定的文件，其他开发的文件在新建分支中

git branch 分支名：创建新分支

git checkout 分支名：切换上传用的分支

git log –oneline:展示提交点

git log:展示提交点的详细信息

git merge 分支a：将a分支合并到当前分支

快速合并：主分支没有更改

三方合并：（更常见）

git diff 文件名.扩展名：查看上次的修改

git commit -am 注释信息：将工作区变动直接加到仓库

git reflog:将所有版本操作都显示出来

git diff 查看尚未暂存的文件更新了哪些部分

git diff filename 　　　　　　　　　　　　　　　　　 查看尚未暂存的某个文件更新了哪些

git diff –cached 　　　　　　 查看已经暂存起来的文件和上次提交的版本之间的差异

git diff –cached filename 　　　　　　　　　　 查看已经暂存起来的某个文件和上次提交的版本之间的差异

git diff 版本号 版本号： 查看某两个版本之间的差异

git diff 版本号 版本号:filename 查看某两个版本的某个文件之间的差异

git checkout 文件名：撤销工作区的修改。

git reset HEAD 文件名：将暂存区的代码撤销到工作区

git diff HEAD – 文件名：对比版本库与工作区的代码差异

git HEAD HEAD^ – 文件名：对比最近两个版本库的某文件差异
